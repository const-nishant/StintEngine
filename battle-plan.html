PHASE 02 Build the Gym Environment 10:30 AM — 12:30 PM
10:30
30 min
Define state space & action space
This is the most important design decision. Keep it simple for Day 1.
  • State (observation): [current_lap, tyre_age, compound_encoded, lap_time_delta, position]
  • Actions: 0 = Stay Out, 1 = Pit Soft, 2 = Pit Medium, 3 = Pit Hard
  • Use gymnasium.spaces.Box for observations, Discrete(4) for actions
  • Normalize all observation values to [0, 1] range
DESIGN
11:00
45 min
Write the Environment class
Create F1StrategyEnv(gymnasium.Env) with the three required methods.
  • reset() — start lap 1, fresh tyres, mid-grid position
  • step(action) — advance one lap, apply tyre deg, update position
  • If action = pit: add 20-25s pit stop time, reset tyre age to 0, change compound
  • Tyre deg model: each lap adds base_deg * (1 + 0.03 * tyre_age) to lap time
  • Reward = position gained this lap (negative = lost positions)
  • Episode ends at lap 57 (Bahrain race length)

Calibrate tyre degradation from real data
Don't hardcode deg values — fit them from the FastF1 data you pulled. This is what makes your sim realistic.
  • Group laps by Compound, compute mean lap time per TyreAge bucket
  • Fit a simple linear or quadratic regression: laptime = a + b*tyre_age + c*tyre_age²
  • Store coefficients per compound (Soft/Medium/Hard)
  • Use these coefficients inside your step() function
  • Quick sanity check: soft should degrade faster than hard

Validate your environment
Before training, make sure the env doesn't crash. Run 10 random episodes manually.
  • Use gymnasium.utils.env_checker.check_env(env) — fixes 90% of bugs
  • Run a random policy loop: env.step(env.action_space.sample())
  • Print state & reward every lap — check they make sense
  • Verify episode terminates at lap 57, not before or after

Set up PPO training
3 lines of actual training code. The complexity is in tuning hyperparams, which you'll iterate on.
  • Wrap env: env = make_vec_env(F1StrategyEnv, n_envs=4) — parallel training
  • Init: model = PPO("MlpPolicy", env, verbose=1, learning_rate=3e-4)
  • Start with model.learn(total_timesteps=50_000) — runs in ~3 min
  • Watch reward in terminal — should trend upward after ~20k steps
from stable_baselines3 import PPO
from stable_baselines3.common.env_util import make_vec_env

env = make_vec_env(F1StrategyEnv, n_envs=4)
model = PPO("MlpPolicy", env, verbose=1)
model.learn(total_timesteps=50_000)
model.save("f1_strategy_agent")
TRAIN
1:50
40 min
Tune reward function
This is where you'll spend real time. If the agent pits every lap or never pits, your reward is wrong. Iterate fast.
  • If agent never pits: add a tyre cliff penalty (if tyre_age > 25, big negative reward)
  • If agent pits every lap: add a pit cost (-25s equivalent in reward units)
  • Add a big end-of-race bonus proportional to finishing position: +50 * (20 - final_pos)
  • Retrain after each reward change — takes only 3 min per run
  • Target: agent should do 2-3 pit stops total across 57 laps

⚑ SURVIVAL TIPS
If FastF1 download stalls, use fastf1.Cache.enable_cache() before any session load
Gym version conflicts are common — pin gymnasium==0.29.1
Start with just 1 compound (Medium only) to validate env, add others after
If reward doesn't improve after 50k steps, your reward scale is off (normalize it)
Use n_envs=4 in make_vec_env — 4x faster training on CPU
Undercut discovered = agent pits 1-2 laps before opponent and gains position
